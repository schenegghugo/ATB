# Presentation packs

Client-side cosmetic packs (art + palette). A pack **never** affects rules — it
only changes what your client draws — so it needs no server authority, no content
hash, and no validation tier (see `ARCHITECTURE.md` §6).

> **Full authoring guide:** [`docs/sprite-packs.md`](../docs/sprite-packs.md) —
> palette-only re-themes, repainting the shipped atlas, hand-authoring rects,
> exporting from TexturePacker / free-tex-packer, and animations. This file is the
> quick in-directory reference.

Point the game at a pack directory with the `ATB_PACK` environment variable:

```bash
ATB_PACK=packs/example ./build-gui/tactical_battler
```

Absent or unloadable ⇒ the built-in primitive renderer (identical to no pack).
Packs theme the **battle** screen — hit *Fight* to see it (the build editor isn't
pack-routed yet). Watch the console for a `Loaded sprite pack …` line on startup.

## Packs in this repo

- **`example/`** — the simplest possible pack: **palette-only**, no atlas. It
  recolours the three tile primitives and leaves everything else built-in — the
  fastest way to re-theme and the smallest thing to copy.
- **`default/`** — a full sprite pack (tiles, unit tokens, a distinct icon per
  spell, plus a champion **cast animation**), generated by
  [`tools/gen_pack.py`](../tools/gen_pack.py) — a zero-dependency, pure-Python
  atlas generator (draws `atlas.png` + emits a matching `pack.json`, so they can't
  drift). Regenerate or fork it: `python3 tools/gen_pack.py [name]`.
- **`example_upscaled_to_96px/`** — the same look at a higher resolution (96 px
  cells instead of 48), a starting point for crisper art.

Both atlas packs make champions flash when they cast — run one and cast a spell.

## Format at a glance

A pack is a directory with a `pack.json` manifest plus any atlas PNGs it names.
Every block is optional and resolves independently, so a pack can restyle just the
walls, or ship **only a palette and no art at all** (like `example/` here).

```jsonc
{
  "schema": 1,
  "name": "My Pack", "version": "1.0.0",
  "tileSize": 32,
  "atlases": { "main": "atlas.png" },       // optional: sprite sheets
  "palette": { "wall": "#464e60" },          // optional: recolour primitives
  "sprites": {                               // optional: atlas sub-rects
    "tiles.wall":       { "atlas": "main", "rect": [0, 0, 32, 32] },
    "units.player":     { "atlas": "main", "rect": [0, 64, 32, 48], "anchor": "bottom" },
    "spells.fireball":  { "atlas": "main", "rect": [0, 96, 32, 32] }
  }
}
```

### Keys the renderer looks up (v1)

| Element          | Sprite key            | Palette key            |
|------------------|-----------------------|------------------------|
| Floor tile       | `tiles.floor`         | `floor`                |
| Wall tile        | `tiles.wall`          | `wall`                 |
| Obstacle tile    | `tiles.obstacle`      | `obstacle`             |
| Player champion  | `units.player`        | *(faction tint)*       |
| Enemy champion   | `units.enemy`         | *(faction tint)*       |
| Summon / object  | `units.<name>` then `spells.<name>` (e.g. `units.brute` → `spells.brute`) | — |
| Spell icon (bar) | `spells.<catalog key>` (e.g. `spells.fireball`) | — |

Resolution ladder per element: **atlas sprite → palette colour → built-in
primitive**. `anchor` is `center` (default) or `bottom`. Ground effects and status
markers aren't pack-routed yet — that wiring is a follow-up.

For the rect/atlas/animation details and packer-export workflow, see
[`docs/sprite-packs.md`](../docs/sprite-packs.md).
